## À propos des applications et des résumés ##

L'AIID est construit sur une base de données de documents et une collection d'applications de navigateur sans serveur. Cela signifie que la base de données est hautement extensible et que la couche d'application peut avoir un nombre illimité de vues dans la base de données. L'intention derrière cette architecture est de permettre à de nombreuses communautés d'avoir des vues différentes sur la base de données. Vous ne devez pas traiter l'AIID comme un produit de données statiques, mais plutôt comme un cadre analytique à partir duquel nous produisons collectivement les meilleurs systèmes intelligents tout en évitant les pires résultats qu'ils pourraient potentiellement produire. N'importe qui peut soumettre une application qui s'interface avec les données d'incident. Accédez au [GitHub Repo](https://github.com/PartnershipOnAI/aiid) pour plus de détails

**Applications vs Résumés**

Un "résumé" dans l'AIID est quelque chose qui génère des statistiques descriptives, des visualisations ou d'autres vues en lecture seule dans la base de données. Les résumés sont censés vivre dans l'environnement de script que vous lisez actuellement. Une "application" est quelque chose qui peut avoir des autorisations d'écriture sur une partie de la base de données. De plus, une application peut maintenir son propre environnement de script isolé de l'environnement de script que vous lisez actuellement.
