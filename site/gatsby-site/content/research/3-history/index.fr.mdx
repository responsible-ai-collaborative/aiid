# Méthodologie de collecte initiale

L'industrie aéronautique aux États-Unis est tenue par la loi de signaler les incidents et les accidents à la Federal Aviation Administration. Il n'y a pas d'exigence de déclaration obligatoire pour les systèmes intelligents, de sorte que l'AIID est construit sur les incidents qui ont acquis une notoriété suffisante pour être couverts dans la presse populaire ou la presse scientifique. Par conséquent, vous devez considérer la base de données comme représentative des "incidents publics", plutôt que de tous les incidents connus pour s'être produits dans l'industrie de l'intelligence artificielle.

La base de données actuelle est dominée par la collection initiale de rapports d'incidents, qui ont été assemblés avec la méthodologie suivante par l'assistant de recherche Sam Yoon,

- Un premier ensemble de liens a été [assemblé par Yampolski et Olsson](/research/2-roadmap).
- Les liens indiquaient des mots clés pour trouver des résultats connexes dans une série de recherches Google.
- Les liens potentiellement pertinents dans les trois premières pages des résultats de recherche Google ont été ouverts et lus pour voir s'ils contiennent des informations de qualité sur l'événement décrit par les sources. Le cas échéant, ils ont été ajoutés au jeu de données.
- D'autres incidents pertinents qui ont été identifiés au cours du processus de recherche ont été inclus dans l'ensemble de données
- Certains types de résultats n'ont pas été inclus parce qu'ils étaient hypothétiques, théoriques et/ou n'ont pas causé de préjudice réel
